Here are all the essential Vite commands for your React + TypeScript YouTube-like app:

üì¶ Basic Vite Commands
Development
bash
# Start development server (default port 5173)
npm run dev

# With custom port
npm run dev -- --port 3000

# Open browser automatically
npm run dev -- --open

# Development with specific host
npm run dev -- --host 0.0.0.0  # Access from other devices

# Force HTTPS in development
npm run dev -- --https

# Combined options
npm run dev -- --port 3000 --open --host
Build
bash
# Production build
npm run build

# Build with preview server
npm run build && npm run preview

# Build with custom mode
npm run build -- --mode staging

# Build with source maps (for debugging)
npm run build -- --sourcemap

# Minify build
npm run build -- --minify

# Build with specific config file
npm run build -- --config vite.config.prod.ts
Preview
bash
# Preview production build locally
npm run preview

# Preview with custom port
npm run preview -- --port 4173

# Preview with specific host
npm run preview -- --host
üöÄ Complete Package.json Scripts
json
{
  "scripts": {
    "dev": "vite",
    "dev:mobile": "vite --port 3000 --host",  // For mobile testing
    "dev:https": "vite --https",  // For PWA testing
    "dev:prod": "vite --mode production",  // Simulate prod
    
    "build": "tsc && vite build",
    "build:prod": "tsc && vite build --mode production",
    "build:staging": "tsc && vite build --mode staging",
    "build:analyze": "tsc && vite build --mode production && npm run analyze",
    "build:no-cache": "tsc && vite build --force",  // Force rebuild
    
    "preview": "vite preview",
    "preview:prod": "vite preview --port 4173",
    "preview:mobile": "vite preview --port 3000 --host",
    
    "lint": "eslint src --ext ts,tsx --report-unused-disable-directives --max-warnings 0",
    "lint:fix": "eslint src --ext ts,tsx --fix",
    "type-check": "tsc --noEmit",
    "test": "vitest",
    "test:watch": "vitest --watch",
    "test:coverage": "vitest --coverage",
    
    "deploy": "npm run build && firebase deploy",  // If using Firebase
    "deploy:gh-pages": "npm run build && gh-pages -d dist",  // GitHub Pages
    
    "analyze": "npx vite-bundle-analyzer",  // Bundle analysis
    "pwa:generate-icons": "pwa-asset-generator public/logo.png public/icons"  // Generate PWA icons
  }
}
üîß Environment-Specific Builds
Create environment files:
text
.env                # Shared (don't commit secrets!)
.env.local          # Local overrides (gitignored)
.env.development    # Development
.env.production     # Production
.env.staging        # Staging
Example .env files:
env
# .env.development
VITE_API_URL=http://localhost:3000/api
VITE_GOOGLE_CLIENT_ID=dev-client-id
VITE_DEBUG=true
env
# .env.production
VITE_API_URL=https://api.your-app.com
VITE_GOOGLE_CLIENT_ID=prod-client-id
VITE_DEBUG=false
Build with specific environment:
bash
# Development build
npm run build -- --mode development

# Production build (default)
npm run build -- --mode production

# Staging build
npm run build -- --mode staging

üö® Troubleshooting Commands

rm -rf node_modules/.vite
rm -rf node_modules/.cache

# Reinstall dependencies
rm -rf node_modules package-lock.json
npm install

# Fix permission issues (Linux/Mac)
chmod -R 755 node_modules

# Check Vite version
npx vite --version

# View all Vite CLI options
npx vite --help
üí° Quick Start Scripts
Create a setup.sh file:

bash
#!/bin/bash
echo "üöÄ Setting up YouTube Clone..."

# Install dependencies
npm install

# Setup environment
cp .env.example .env.local

# Generate PWA icons
npm run pwa:generate-icons

# Start development server
npm run dev:mobile

# Open in browser (Mac)
open http://localhost:3000
Make it executable:

bash
chmod +x setup.sh
./setup.sh
